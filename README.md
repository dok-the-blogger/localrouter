# Настройка обхода блокировок на роутере Asus (Entware)

Инструкция для восстановления конфигурации Xray + SSH-туннеля после сброса роутера или замены железа.

## 🖥️ Спецификация стенда (Environment)
Конфигурация отлажена и гарантированно работает на следующем стеке:
- **Железо:** Asus RT-AX82U (аппаратная ревизия V2)
- **Процессор:** ARMv7 (Broadcom BCM675x, Cortex A7, 3 ядра) — *важно при ручном скачивании бинарников Xray (нужен linux-arm32-v7a).*
- **Прошивка:** GNUton's Asuswrt-Merlin (build 3004.388.11_1-gnuton0_beta1). 
*Внимание: Наличие прошивки Merlin или её форка критически важно! Именно она обеспечивает работу пользовательских скриптов в разделе /jffs/, которые инициализируют флешку с Entware при загрузке.*

---

## 1. Установка пакетов
Необходимо установить следующие пакеты через консоль (Entware должна быть уже развернута через amtm):

```bash
opkg install autossh openssh-client curl
```

Если Xray не установлен вручную, его тоже нужно установить (например, из репозитория).

## 2. Настройка окружения
Перед запуском автоматического деплоя необходимо подготовить секреты.

### 2.1 Файл переменных окружения
Создайте файл `/opt/etc/.router.env` и добавьте в него следующие переменные:

```bash
SELECTEL_IP="x.x.x.x"
SECRET_PASS="your_password"
DO_IP="x.x.x.x"
TARGETS="192.168.50.112 192.168.50.59"
```

*   `SELECTEL_IP` — IP адрес вашего сервера Selectel.
*   `SECRET_PASS` — Пароль для Shadowsocks/VLESS.
*   `DO_IP` — IP адрес вашего дроплета DigitalOcean.
*   `TARGETS` — Список IP-адресов устройств в локальной сети, трафик которых нужно проксировать (через пробел).

### 2.2 SSH ключ
Скопируйте приватный ключ `do_ss` в `/opt/etc/ssh/`. Это действие выполняется вручную из соображений безопасности.
**ВАЖНО:** Жестко ограничьте права на ключ, иначе SSH откажется работать:
```bash
chmod 600 /opt/etc/ssh/do_ss
```

## 3. Деплой
Репозиторий на роутере: `/tmp/mnt/router_disk/entware/root/repo/`
```bash
cd /tmp/mnt/router_disk/entware/root/repo
./deploy.sh
```

Скрипт автоматически:
1.  Подтянет свежие изменения из репозитория.
2.  Подставит переменные из `.router.env` в конфигурационные файлы.
3.  Разложит конфиги и скрипты по нужным путям (`/opt/etc/...`, `/jffs/scripts/...`).
4.  Выставит права на выполнение.
5.  Перезапустит сервис Xray и обновит правила фаервола.

## 4. JFFS
Убедитесь, что в веб-интерфейсе роутера (Administration -> System) включена опция **"Enable JFFS custom scripts and configs"**.

## 5. Ad-Blocking (Xray Blackhole)
Xray используется не только для обхода блокировок, но и как сетевой санитар.
В конфигурации реализована фильтрация рекламного и трекингового трафика на уровне доменов.
Подробнее о логике работы и приоритетах правил читайте в [docs/XRAY_CONFIG.md](docs/XRAY_CONFIG.md).
